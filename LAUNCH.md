## LinkedIn
Most automation tools break the moment an app has no API. So I built Camel AI to work like a human instead — it drives the actual UI.

Give it a goal in plain English and Camel AI:
→ tests any web/desktop app and produces a ranked fix-brief an AI coder can act on
→ automates real workflows (medical billing, finance, marketing…)
→ pauses for you on logins/2FA, then continues
→ runs scheduled jobs unattended

The only API it ever needs is an LLM API (a free Gemini key works). Open source, one-line install.

## Hacker News (Show HN)
Title: Show HN: Camel AI – an AI that tests and automates any app through its UI

Body:
Camel AI drives software the way a person does — clicking, typing, reading the
screen — so it works on apps that have no API (or one you can't access). Two
uses: (1) testing — it clicks every control, finds dead buttons / console errors
/ a11y issues, and emits a machine-readable fix-brief for a coding agent; (2)
automation — plain-English goals, with a human handoff for 2FA/login.

Three drivers (web via Playwright, Windows desktop via UI Automation, vision
fallback), an MCP mode that runs on your Claude subscription (no key), a web
dashboard, and scheduled autonomous jobs. Free Gemini tier by default.

Repo: https://github.com/DilawarShafiq/camel-ai — happy to answer anything.
